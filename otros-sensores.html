Otros sensores: del “¿hay luz?” al “¿dónde estoy en el espacio?”

Ya leíste entradas digitales y análogas. Ahora verás sensores típicos de proyectos reales. La idea no es memorizar modelos, sino entender patrones: qué miden, qué necesitan (tiempos, librerías, voltaje) y cómo traducir sus datos a decisiones.

1) Patrones (para no volverte coleccionista de módulos)

Muchos sensores caen en una de estas familias:

  • Digital simple: HIGH/LOW (PIR, switch, algunos sensores de flama).
  • Digital por protocolo: I2C / SPI / OneWire (IMU, pantallas, memorias).
  • Por tiempo: miden pulsos (ultrasonido HC-SR04).
  • Análogo: voltaje variable (potenciómetro, LDR, algunos sensores de gas).
Idea conservadora que salva proyectos: antes de “integrar”, haz un sketch mínimo que solo lea el sensor y lo imprima por Serial. Primero entender, luego combinar.

2) Ultrasonido HC-SR04: distancia por tiempo

El HC-SR04 manda un “clic” ultrasónico y mide cuánto tarda en rebotar. Arduino mide el tiempo del pulso de regreso y lo convierte a distancia.

Pin móduloConectar aNota
VCC5VEn UNO es directo. En placas 3.3V revisar compatibilidad.
GNDGNDSiempre GND común.
TRIGD9 (ejemplo)Salida: manda pulso.
ECHOD10 (ejemplo)Entrada: recibe pulso.
 
// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Sensor: Ultrasonido HC-SR04 (distancia)
// Autoría académica: Dr. Iroel Heredia (2026)
// ------------------------------------------------------------

const int trigPin = 9;
const int echoPin = 10;

void setup() {
  pinMode(trigPin, OUTPUT);
  pinMode(echoPin, INPUT);
  Serial.begin(9600);
}

void loop() {
  // 1) Asegurar trig en LOW un momento
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);

  // 2) Pulso de 10 microsegundos para disparar
  digitalWrite(trigPin, HIGH);
  delayMicroseconds(10);
  digitalWrite(trigPin, LOW);

  // 3) Medir duración del pulso ECHO (en microsegundos)
  long duracion = pulseIn(echoPin, HIGH, 30000); // timeout 30 ms (~5 m)

  // 4) Convertir a cm.
  // Velocidad del sonido ~343 m/s => 0.0343 cm/us
  // La señal va y regresa, por eso /2
  float distanciaCm = (duracion * 0.0343) / 2.0;

  Serial.print("Duracion(us): ");
  Serial.print(duracion);
  Serial.print("  Distancia(cm): ");
  Serial.println(distanciaCm);

  delay(100);
}
Notas reales: superficies blandas (tela), ángulos raros o espacios abiertos pueden dar lecturas inestables. Esto no es “falla del Arduino”, es física con carácter.

3) PIR: detección de movimiento (digital)

El PIR detecta cambios de radiación infrarroja (cuerpos calientes moviéndose). Normalmente entrega una salida digital: HIGH cuando detecta movimiento. 

// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Sensor: PIR (movimiento) -> LED
// Autoría académica: Dr. Iroel Heredia (2026)
// ------------------------------------------------------------

const int pirPin = 2;
const int ledPin = LED_BUILTIN;

void setup() {
  pinMode(pirPin, INPUT);
  pinMode(ledPin, OUTPUT);
  Serial.begin(9600);

  // Algunos PIR necesitan "calentamiento" 30-60s al iniciar
  Serial.println("PIR iniciando... espera un momento.");
  delay(2000);
}

void loop() {
  int m = digitalRead(pirPin);

  if (m == HIGH) {
    digitalWrite(ledPin, HIGH);
    Serial.println("Movimiento detectado");
  } else {
    digitalWrite(ledPin, LOW);
  }

  delay(50);
}

4) DHT11 / DHT22: temperatura y humedad (con librería)

Estos sensores entregan datos por un protocolo digital. En clase, lo práctico es usar librería. DHT11 es más económico y menos preciso; DHT22 suele ser más preciso.

Nota: Para compilar este ejemplo necesitas instalar una librería de DHT (por el Library Manager). Si algo falla, empieza por lo tradicional: cableado + tipo de sensor correcto + pin correcto. 
// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Sensor: DHT11/DHT22 (temperatura y humedad)
// Autoría académica: Dr. Iroel Heredia (2026)
//
// Requiere librería "DHT sensor library" (Adafruit) y "Adafruit Unified Sensor"
// ------------------------------------------------------------

#include <DHT.h>

#define DHTPIN  2     // pin de datos
#define DHTTYPE DHT11 // cambia a DHT22 si usas ese modelo

DHT dht(DHTPIN, DHTTYPE);

void setup() {
  Serial.begin(9600);
  dht.begin();
}

void loop() {
  float h = dht.readHumidity();
  float t = dht.readTemperature(); // °C por defecto

  if (isnan(h) || isnan(t)) {
    Serial.println("Error leyendo DHT");
    delay(1000);
    return;
  }

  Serial.print("Humedad: ");
  Serial.print(h);
  Serial.print("%  Temp: ");
  Serial.print(t);
  Serial.println(" C");

  delay(1500);
}

5) Sensor capacitivo “touch”: tocar sin botón

Un sensor capacitivo detecta cambios de capacitancia cuando acercas el dedo. Muchos módulos entregan una salida digital (HIGH/LOW) como si fuera botón. 

// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Sensor: Touch capacitivo (salida digital) -> cambia estado de LED
// Autoría académica: Dr. Iroel Heredia (2026)
//
// Nota: este ejemplo es como "evento de toque" (con debounce simple)
// ------------------------------------------------------------

const int touchPin = 2;       // salida digital del módulo
const int ledPin   = 13;

bool encendido = false;
int  ultimo = LOW;
unsigned long t0 = 0;
const unsigned long debounceMs = 40;

void setup() {
  pinMode(touchPin, INPUT);
  pinMode(ledPin, OUTPUT);
}

void loop() {
  int v = digitalRead(touchPin);

  if (v != ultimo) { t0 = millis(); ultimo = v; }

  if ((millis() - t0) > debounceMs) {
    static bool listo = true;

    if (v == HIGH && listo) {
      encendido = !encendido;
      digitalWrite(ledPin, encendido ? HIGH : LOW);
      listo = false;
    }
    if (v == LOW) listo = true;
  }
}

6) IMU (MPU-6050): acelerómetro + giroscopio (I2C)

Una IMU mide aceleración (incluida gravedad) y velocidad angular. Se comunica por I2C, que usa dos líneas: SDA y SCL.

SeñalUNONota
SDAA4Línea de datos I2C
SCLA5Línea de reloj I2C
Realidad práctica: En I2C, el cableado y el ruido importan. Cables largos = lecturas raras. Mantén conexiones cortas para empezar. 
// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Sensor: MPU-6050 (I2C) lectura básica (registro WHO_AM_I)
// Autoría académica: Dr. Iroel Heredia (2026)
//
// Este ejemplo verifica que el dispositivo responde en I2C.
// Para lecturas completas (acel/gyro) normalmente se usa librería.
// ------------------------------------------------------------

#include <Wire.h>

const byte MPU_ADDR = 0x68;   // dirección común del MPU-6050
const byte WHO_AM_I = 0x75;   // registro de identificación

void setup() {
  Serial.begin(9600);
  Wire.begin();

  // "Despertar" el MPU-6050 (sale de sleep)
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(0x6B); // PWR_MGMT_1
  Wire.write(0);    // set to 0 wakes up
  Wire.endTransmission(true);

  Serial.println("MPU-6050: iniciando...");
}

void loop() {
  // Leer WHO_AM_I
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(WHO_AM_I);
  Wire.endTransmission(false);
  Wire.requestFrom(MPU_ADDR, (byte)1, (byte)true);

  if (Wire.available()) {
    byte id = Wire.read();
    Serial.print("WHO_AM_I = 0x");
    Serial.println(id, HEX);
  } else {
    Serial.println("No respuesta I2C (revisa SDA/SCL, GND, VCC)");
  }

  delay(1000);
}

7) Buenas prácticas (que parecen aburridas hasta que salvan el día)

  • Serial primero: imprime valores antes de “hacer magia” con LEDs/motores.
  • Calibración: umbrales dependen del ambiente (luz, temperatura, distancia).
  • Tiempo: algunos sensores requieren esperar (PIR) o intervalos (DHT).
  • Voltajes: revisa si el módulo es 5V o 3.3V (sobre todo con sensores modernos).
  • GND común: si compartes señal, comparte tierra.
Ejercicio integrador: Con un HC-SR04, prende un buzzer si alguien está a menos de 30 cm, y enciende un LED si está entre 30–80 cm. Usa if / else if.
