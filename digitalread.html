digitalRead(): leer un botón y tomar decisiones

digitalRead(pin) te devuelve HIGH o LOW según el estado de un pin configurado como entrada. Es la base para botones, switches y sensores “sí/no”. Hoy usaremos un botón para: (1) encender un LED, y (2) cambiar de LED 1→2→3→4 con cada pulsación, apagando los anteriores.

0) Cableado recomendado (simple y confiable)

Vamos a usar el pull-up interno de Arduino. Es decir:

  • El pin del botón se configura como INPUT_PULLUP.
  • Un lado del botón va al pin (por ejemplo D2).
  • El otro lado del botón va a GND.
Con INPUT_PULLUP, el botón se lee al revés de lo “intuitivo”: sin presionar = HIGH, presionado = LOW. No es brujería: es electrónica ordenada.

1) Ejemplo 1: Un botón enciende un LED (mientras lo mantengo presionado)

Este es el comportamiento más directo: si el botón está presionado, enciendo el LED; si no, lo apago. No “recuerda” estados: solo reacciona. 

// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Tema: digitalRead() — Botón que enciende un LED
// Autoría académica: Dr. Iroel Heredia (2026)
// Cableado recomendado:
//   - Botón entre D2 y GND
//   - LED (con resistencia) en D13 o usar LED_BUILTIN
// Nota: INPUT_PULLUP => suelto = HIGH, presionado = LOW
// ------------------------------------------------------------

const int boton = 2;              // Pin del botón
const int led   = LED_BUILTIN;    // LED integrado (o cambia a un pin como 13)

void setup() {
  pinMode(boton, INPUT_PULLUP);   // Activamos pull-up interno
  pinMode(led, OUTPUT);           // LED como salida
}

void loop() {
  int estado = digitalRead(boton);   // Leemos el botón

  // Como usamos INPUT_PULLUP:
  // - Presionado => LOW
  // - Suelto     => HIGH
  if (estado == LOW) {
    digitalWrite(led, HIGH);         // Botón presionado => LED encendido
  } else {
    digitalWrite(led, LOW);          // Botón suelto => LED apagado
  }
}

2) Antes del ejemplo 2: ¿Por qué a veces “cuenta doble”? (debounce)

Un botón mecánico no hace un cambio perfecto de ON a OFF: al presionar, los contactos “rebotan” milésimas de segundo y se generan varias transiciones rápidas. Para el ojo humano es una sola pulsación, pero para Arduino pueden ser varias. Por eso usamos antirrebote (debounce).

Aquí implementamos un debounce por tiempo usando millis() (sin congelar el programa con delays largos).

3) Ejemplo 2: Cada pulsación cambia el LED 1→2→3→4 (apagando los anteriores)

Este ejemplo ya “recuerda” en qué LED va. Cada vez que detecta una pulsación válida, avanza al siguiente LED. Solo uno queda encendido a la vez: al encender el nuevo, apaga los demás.

LED Pin sugerido Nota
LED 1D3Con resistencia en serie (ej. 220Ω–330Ω)
LED 2D4Con resistencia
LED 3D5Con resistencia
LED 4D6Con resistencia
 
// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Tema: digitalRead() — Botón que cambia LED 1→2→3→4 (solo uno encendido)
// Autoría académica: Dr. Iroel Heredia (2026)
//
// Cableado recomendado:
//   - Botón entre D2 y GND
//   - LEDs (con resistencias) en D3, D4, D5, D6 hacia GND
//
// Nota: INPUT_PULLUP => suelto = HIGH, presionado = LOW
// Incluye debounce (antirrebote) por tiempo con millis()
// ------------------------------------------------------------

const int boton = 2;

// LEDs (puedes cambiar pines si lo necesitas)
const int led1 = 3;
const int led2 = 4;
const int led3 = 5;
const int led4 = 6;

// Variables para el debounce
int ultimoEstadoBoton = HIGH;       // Estado previo leído (con pull-up inicia HIGH)
unsigned long ultimoCambio = 0;     // Tiempo del último cambio
const unsigned long debounceMs = 40; // Ventana típica 30–60 ms

// Variable de “memoria”: qué LED está activo (0..3)
int indiceLed = 0;

void setup() {
  pinMode(boton, INPUT_PULLUP);

  pinMode(led1, OUTPUT);
  pinMode(led2, OUTPUT);
  pinMode(led3, OUTPUT);
  pinMode(led4, OUTPUT);

  // Encendemos el primer LED al inicio (opcional)
  encenderSolo(indiceLed);
}

void loop() {
  int lectura = digitalRead(boton);

  // Si la lectura cambió respecto a la última lectura registrada, reiniciamos temporizador
  if (lectura != ultimoEstadoBoton) {
    ultimoCambio = millis();
    ultimoEstadoBoton = lectura;
  }

  // Si ya pasó el tiempo de debounce, aceptamos la lectura como estable
  if ( (millis() - ultimoCambio) > debounceMs ) {

    // Queremos detectar la “pulsación” como evento (no mantener presionado):
    // Con pull-up: presionado = LOW.
    // Entonces, el evento será cuando esté en LOW.
    //
    // Para que no cuente múltiples veces mientras lo mantienes presionado,
    // esperamos a que suelte (HIGH) antes de permitir otro conteo.
    //
    // Implementamos un “candado” muy simple con una variable estática.
    static bool listoParaContar = true;

    if (lectura == LOW && listoParaContar) {
      // Evento: se presionó el botón (pulsación válida)
      indiceLed = indiceLed + 1;

      // Si se pasó de 3, regresa a 0 (ciclo 1→2→3→4→1…)
      if (indiceLed > 3) {
        indiceLed = 0;
      }

      encenderSolo(indiceLed);

      // Bloqueamos conteo hasta que suelte el botón
      listoParaContar = false;
    }

    if (lectura == HIGH) {
      // Si ya soltó, habilitamos el conteo para la próxima pulsación
      listoParaContar = true;
    }
  }
}

// Función auxiliar: apaga todos y enciende solo el LED indicado
void encenderSolo(int idx) {
  // Apagar todos
  digitalWrite(led1, LOW);
  digitalWrite(led2, LOW);
  digitalWrite(led3, LOW);
  digitalWrite(led4, LOW);

  // Encender solo el seleccionado
  if (idx == 0) digitalWrite(led1, HIGH);
  if (idx == 1) digitalWrite(led2, HIGH);
  if (idx == 2) digitalWrite(led3, HIGH);
  if (idx == 3) digitalWrite(led4, HIGH);
}
Qué aprender aquí:
1) El botón es una entrada digital (HIGH/LOW).
2) “Pulsación” es un evento, no un estado sostenido.
3) Debounce evita dobles conteos (la realidad vibra; el código pone orden).
4) Una variable (indiceLed) funciona como memoria simple.

Errores típicos

- Olvidar que INPUT_PULLUP invierte lógica
- No compartir GND (botón y LEDs deben tener referencia común)
- No usar resistencia en LEDs
- Contar varias veces por rebote (debounce)

Tip rápido

Si tu botón “no sirve”, 70% de las veces es el cableado. El otro 30%… también, pero con más confianza.
