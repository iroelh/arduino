Otros actuadores: cuando el “músculo” ya no cabe en un pin

LEDs fueron el calentamiento. Aquí entran actuadores que requieren más “fuerza” o control: servo (posición), motor DC (velocidad), relevador (conmutar cargas), y buzzer (sonido). También veremos por qué el Arduino no debe “hacer de cargador universal”.

1) Principio general: Arduino decide, el driver ejecuta

Un pin digital es excelente para señal, pero malo para potencia. Muchos actuadores piden más corriente de la que un pin puede entregar de forma segura. Por eso existe el “músculo externo”: transistores, MOSFETs, drivers, relés.

Advertencia sana: no alimentes motores/bombas/relés grandes desde el pin de Arduino. Si lo haces, a veces funciona… justo hasta que deja de funcionar. Y el humo sale sin pedir permiso.

Necesitas driver / MOSFET cuando…

La carga consume “mucho” (motores, tiras LED largas, bombas) o es inductiva.

Necesitas relevador cuando…

Quieres conmutar algo de mayor voltaje/corriente (AC o DC) sin mezclar potencias en el Arduino.

2) Servo: controlar posición (0–180°)

Un servo típico (SG90, MG90S, etc.) se controla por una señal tipo PWM especial (pulsos). Lo más fácil es usar la librería Servo.h.

  • Servo tiene 3 cables: GND, Vcc (5V), señal.
  • La señal puede ir a un pin digital (ej. D9).
  • Si el servo se pone “nervioso” o reinicia Arduino, aliméntalo con fuente externa y GND común.
// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Tema: Actuador Servo — posición con potenciómetro
// Autoría académica: Dr. Iroel Heredia (2026)
//
// Cableado:
//   - Potenciómetro: centro a A0, extremos a 5V y GND
//   - Servo: señal a D9, Vcc a 5V (o fuente externa), GND a GND
// ------------------------------------------------------------

#include <Servo.h>

Servo miServo;

const int potPin   = A0;
const int servoPin = 9;

void setup() {
  miServo.attach(servoPin);
}

void loop() {
  int v = analogRead(potPin);              // 0..1023
  int angulo = map(v, 0, 1023, 0, 180);    // 0..180 grados

  miServo.write(angulo);
  delay(15); // pequeñísima pausa para estabilidad
}

3) Motor DC: velocidad por PWM (con MOSFET o driver)

Un motor DC no se conecta directo al pin. Normalmente lo controlas con:

  • Un MOSFET (control eficiente de velocidad por PWM) + diodo flyback.
  • O un driver puente H (si quieres invertir giro: L298N, TB6612FNG, etc.).

Aquí mostramos el control conceptual de velocidad (la parte de Arduino). El hardware de potencia es obligatorio. 

// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Tema: Motor DC — velocidad con PWM (requiere MOSFET/driver)
// Autoría académica: Dr. Iroel Heredia (2026)
//
// NOTA: No conectes el motor directo al pin.
// Usa MOSFET + diodo flyback + fuente externa + GND común.
// ------------------------------------------------------------

const int potPin = A0;
const int motorPWM = 9; // pin PWM hacia el gate/entrada del driver

void setup() {
  pinMode(motorPWM, OUTPUT);
}

void loop() {
  int v = analogRead(potPin);              // 0..1023
  int pwm = map(v, 0, 1023, 0, 255);       // 0..255
  analogWrite(motorPWM, pwm);              // “más PWM” => más velocidad (típicamente)
  delay(10);
}
Truco didáctico: PWM es como “dar empujoncitos” muy rápido. El motor promedia esos empujes y parece una velocidad continua.

4) Relevador: encender/apagar cargas mayores (con cuidado)

Un relevador (relay) es un interruptor controlado eléctricamente. Te sirve para prender/apagar algo de mayor potencia (una lámpara, un ventilador, etc.). Lo correcto es usar un módulo de relevador (ya trae transistor y diodo).

Seguridad: si vas a conmutar AC (corriente de pared), eso ya no es “tarea de salón sin supervisión”. Trabaja con personal capacitado y medidas de seguridad. Arduino no es un casco. 
// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Tema: Relevador — encendido/apagado con digitalWrite
// Autoría académica: Dr. Iroel Heredia (2026)
//
// Usar módulo de relevador:
//   IN  -> pin digital (ej. 7)
//   VCC -> 5V
//   GND -> GND
//
// Nota: algunos módulos son "activo en LOW".
// Si al revés, invierte HIGH/LOW.
// ------------------------------------------------------------

const int rele = 7;

void setup() {
  pinMode(rele, OUTPUT);
}

void loop() {
  digitalWrite(rele, HIGH); // encender
  delay(1000);
  digitalWrite(rele, LOW);  // apagar
  delay(1000);
}

5) Buzzer: sonido (simple y rápido)

Un buzzer puede ser:

  • Activo: suena con solo encenderlo (es más “plug & play”).
  • Pasivo: necesita una frecuencia (para notas). Arduino tiene tone().
// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Tema: Buzzer pasivo — tone()
// Autoría académica: Dr. Iroel Heredia (2026)
//
// Cableado (simple):
//   buzzer -> pin 8 y GND (según módulo, puede requerir resistencia/transistor)
// ------------------------------------------------------------

const int buzzer = 8;

void setup() {
  // nada especial
}

void loop() {
  tone(buzzer, 440);   // La 440 Hz
  delay(300);
  noTone(buzzer);
  delay(300);

  tone(buzzer, 660);   // Mi 660 Hz
  delay(300);
  noTone(buzzer);
  delay(600);
}

6) Retos (para práctica)

  • Reto Servo: mueve el servo a 0°, 90°, 180° con tres botones.
  • Reto Motor DC: potenciómetro controla velocidad, y un botón hace “stop” inmediato.
  • Reto Relevador: con un LDR, prende una lámpara cuando oscurezca (umbral + histéresis).
  • Reto Buzzer: crea un “código morse” básico: punto = 200ms, raya = 600ms.
Cierre: Los actuadores “serios” te obligan a pensar en energía. Es un excelente hábito: quien diseña interacción también diseña responsabilidades.

Regla conservadora

Si un actuador “jala fuerte”, se trata con respeto: fuente externa, diodo, y GND común. Tradición eléctrica: no pelearse con Ohm.

Tip rápido

Si algo se resetea “misteriosamente”, casi siempre es alimentación. El misterio es solo un multímetro que aún no conoces.
