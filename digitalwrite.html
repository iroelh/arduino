digitalWrite(): mandar un “sí/no” al mundo físico

digitalWrite(pin, value) sirve para poner un pin digital en HIGH o LOW. Traducción: “enciende” o “apaga” una salida. Es la forma más directa de controlar un LED, activar un relevador, o decirle a un driver que “ahora sí, mueve el músculo”.

1) ¿Qué hace digitalWrite?

Piensa en un pin digital como una pequeña puerta que solo sabe decir dos cosas: 0 (LOW) o 1 (HIGH). Cuando llamas digitalWrite, Arduino aplica ese estado al pin.

HIGH

Estado “encendido”. Normalmente equivale a ~5V en UNO (o ~3.3V en algunas placas).

LOW

Estado “apagado”. Equivale a 0V (tierra / GND).

Importante: Para usar un pin como salida debes declararlo primero con pinMode(pin, OUTPUT) dentro de setup(). Si no, es como querer hablar… con la boca cerrada.

2) Ejemplo base: Blink (tuneado para IBERO)

Este es el “hola mundo” de Arduino. Usaremos el LED integrado en la mayoría de placas: LED_BUILTIN. En placas tipo UNO suele ser el pin 13. 

// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Tema: digitalWrite() — Ejemplo Blink
// Autoría académica: Dr. Iroel Heredia (2026)
// ------------------------------------------------------------

void setup() {
  // En setup() configuramos lo que solo debe ocurrir UNA vez al iniciar.
  pinMode(LED_BUILTIN, OUTPUT);   // El LED interno será una SALIDA
}

void loop() {
  // En loop() repetimos la acción una y otra vez (como un pequeño “reflejo”).
  digitalWrite(LED_BUILTIN, HIGH);  // Encender LED
  delay(1000);                      // Esperar 1000 ms (1 segundo)
  digitalWrite(LED_BUILTIN, LOW);   // Apagar LED
  delay(1000);                      // Esperar 1 segundo
}

3) Desafío guiado: 2 segundos encendido y 2 apagado (explicando delay)

delay(ms) detiene el programa por la cantidad de milisegundos indicada. 1000 ms = 1 segundo. Por eso delay(2000) equivale a 2 segundos.

Ojo docente: delay es perfecto para empezar (y ver resultados rápidos). Más adelante, cuando quieras “hacer varias cosas a la vez”, aprenderás a evitarlo.

Tu reto aquí es modificar Blink para que:

  • El LED esté encendido 2 segundos
  • El LED esté apagado 2 segundos

4) Encender 4 LEDs al mismo tiempo (dos formas)

Para este ejercicio necesitas 4 LEDs externos, cada uno con su resistencia, conectados a cuatro pines digitales distintos (y a GND).

4.1 Forma A: usando números de pin directamente

Esta forma es simple, pero puede volverse difícil de mantener cuando el proyecto crece. 

// 4 LEDs a la vez (Forma A): usando números de pin directamente

void setup() {
  pinMode(2, OUTPUT);
  pinMode(3, OUTPUT);
  pinMode(4, OUTPUT);
  pinMode(5, OUTPUT);
}

void loop() {
  digitalWrite(2, HIGH);
  digitalWrite(3, HIGH);
  digitalWrite(4, HIGH);
  digitalWrite(5, HIGH);

  delay(1000);

  digitalWrite(2, LOW);
  digitalWrite(3, LOW);
  digitalWrite(4, LOW);
  digitalWrite(5, LOW);

  delay(1000);
}

4.2 Forma B: usando variables con nombres de perritos

Aquí nombramos los pines con variables para que el código sea más legible. (Sí, los perritos ayudan a que el código no muerda.) 

// 4 LEDs a la vez (Forma B): usando variables con nombres de perritos

const int firulais = 2;  // LED 1
const int luna     = 3;  // LED 2
const int coco     = 4;  // LED 3
const int max      = 5;  // LED 4

void setup() {
  pinMode(firulais, OUTPUT);
  pinMode(luna, OUTPUT);
  pinMode(coco, OUTPUT);
  pinMode(max, OUTPUT);
}

void loop() {
  // Encender todos
  digitalWrite(firulais, HIGH);
  digitalWrite(luna, HIGH);
  digitalWrite(coco, HIGH);
  digitalWrite(max, HIGH);

  delay(1000);

  // Apagar todos
  digitalWrite(firulais, LOW);
  digitalWrite(luna, LOW);
  digitalWrite(coco, LOW);
  digitalWrite(max, LOW);

  delay(1000);
}
Consejo clásico: Si cambias un pin, con variables solo lo cambias en un lugar. Con números “duros”, lo cambias en 10… y en 2 se te olvida.

5) Desafíos (sin solución, para que los alumnos lo resuelvan)

Desafío 1: Encender solo LED 1 + 3 y alternar con LED 2 + 4

Objetivo: hacer un patrón alternado (como “pares e impares”):

  • Primero: LED 1 y LED 3 encendidos, LED 2 y LED 4 apagados
  • Luego: LED 2 y LED 4 encendidos, LED 1 y LED 3 apagados
  • Repetir indefinidamente

Pista: piensa en dos “grupos” y alterna con delay. Usa variables (perritos) para que se entienda.

Desafío 2: Secuencia 1→2→3→4 y regreso 4→3→2→1 (apagando el anterior)

Objetivo: un “scanner” de LEDs (como luces que van y regresan):

  • Enciende LED 1 (solo ese)
  • Luego LED 2, pero apaga LED 1
  • Luego LED 3, apaga LED 2
  • Luego LED 4, apaga LED 3
  • Regresa: LED 3 (apaga LED 4), luego LED 2 (apaga LED 3), luego LED 1 (apaga LED 2)
  • Repite

Pista: el “apagado del anterior” es parte del estilo del ejercicio. No es “acumular”, es “pasar la estafeta”.

Errores típicos

- Olvidar pinMode(..., OUTPUT)
- Conectar LED sin resistencia
- Olvidar GND común
- Usar un cable “solo carga” y culpar al universo

Tip de estilo

Si un alumno nombra sus pines “firulais/luna/coco/max” y el código queda claro, la ingeniería también sonríe (en silencio, pero sonríe).
