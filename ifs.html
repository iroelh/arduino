If: el “cerebro” de tu proyecto (decidir qué hacer y cuándo)

Sensores te dan datos, actuadores hacen cosas… y los if deciden. En Arduino, programar casi siempre es traducir el mundo a reglas: si pasa esto, hago aquello. Hoy vemos if, else, else if, operadores (&&, ||, !) y if anidados.

1) If básico (una sola condición)

Un if ejecuta un bloque solo si la condición es verdadera: 

if (condicion) {
  // Esto pasa SOLO si condicion es verdadera
}

En Arduino, “condición” suele venir de digitalRead o comparaciones con analogRead.

2) If + else (dos caminos)

Si la condición se cumple, ejecuta un bloque; si no, ejecuta el otro: 

if (condicion) {
  // Camino A
} else {
  // Camino B
}

3) else if (varios niveles)

Cuando tienes varios casos, usas una “escalera”: 

if (cond1) {
  // Caso 1
} else if (cond2) {
  // Caso 2
} else if (cond3) {
  // Caso 3
} else {
  // Si no fue ninguno de los anteriores
}
Regla de oro: el orden importa. Arduino evalúa de arriba hacia abajo. El primer caso verdadero “gana” y ya no revisa los de abajo.

4) Comparadores y lógica

Estos son los símbolos que vas a usar todo el tiempo:

Operador Significa Ejemplo
==igual aif (x == 10)
!=distinto deif (x != 0)
<menor queif (x < 500)
<=menor o igualif (x <= 500)
>mayor queif (x > 700)
>=mayor o igualif (x >= 700)
Operador Se lee como Ejemplo
&& Y (AND): ambas deben ser verdaderas if (a > 10 && b < 5)
|| O (OR): con que una sea verdadera if (a == 0 || b == 0)
! NO (NOT): invierte la condición if (!encendido)

5) If anidados (cuando una decisión depende de otra)

Un if anidado es un if dentro de otro. Útil cuando primero filtras un caso grande, y luego eliges un caso pequeño. 

if (modo == 1) {
  if (sensor > 700) {
    // Caso específico dentro del modo 1
  } else {
    // Otro caso dentro del modo 1
  }
} else {
  // Si no estás en modo 1
}
Tip de claridad: si tienes más de 2 niveles de anidación, probablemente ya te pide a gritos organizarlo en funciones o replantear con else if. Si el código parece laberinto, el proyecto también.

6) Ejemplo rápido en Arduino: botón decide dos acciones

Con pull-up interno, presionado = LOW. Si presiono, enciendo LED; si no, lo apago. (Ya lo hiciste, pero aquí se ve como “decisión” explícita.) 

// Botón (INPUT_PULLUP) decide si enciende LED

const int boton = 2;
const int led   = LED_BUILTIN;

void setup() {
  pinMode(boton, INPUT_PULLUP);
  pinMode(led, OUTPUT);
}

void loop() {
  int e = digitalRead(boton);

  if (e == LOW) {
    digitalWrite(led, HIGH);
  } else {
    digitalWrite(led, LOW);
  }
}

7) Ejemplo con analogRead: semáforo por rangos (else if)

Imagina un potenciómetro en A0 y tres LEDs (verde/amarillo/rojo). Según el valor, encendemos uno u otro. Esto es un ejemplo clásico de “rangos”. 

// ------------------------------------------------------------
// Curso Arduino — Universidad Iberoamericana (IBERO)
// Tema: If / else if — Semáforo por rangos con potenciómetro
// Autoría académica: Dr. Iroel Heredia (2026)
//
// Potenciómetro a A0
// LEDs: verde=3, amarillo=4, rojo=5 (con resistencias)
// ------------------------------------------------------------

const int pot = A0;
const int verde = 3;
const int amarillo = 4;
const int rojo = 5;

void setup() {
  pinMode(verde, OUTPUT);
  pinMode(amarillo, OUTPUT);
  pinMode(rojo, OUTPUT);
}

void loop() {
  int v = analogRead(pot);  // 0..1023

  // Limpiamos estado
  digitalWrite(verde, LOW);
  digitalWrite(amarillo, LOW);
  digitalWrite(rojo, LOW);

  // Rangos (ajusta a gusto)
  if (v < 341) {
    digitalWrite(verde, HIGH);
  } else if (v < 682) {
    digitalWrite(amarillo, HIGH);
  } else {
    digitalWrite(rojo, HIGH);
  }
}

8) Ejemplo con if anidado: “modo manual” vs “modo automático”

Este ejemplo ilustra una situación típica: un botón cambia de modo. En modo manual decides con potenciómetro; en modo automático decides por luz (LDR). (No lo ejecutas tal cual si no tienes todo cableado; es para entender estructura mental.) 

// Ejemplo conceptual de if anidado (modo manual / automático)

bool modoManual = true; // Imagina que esto cambia con un botón

int lecturaPot = 0;     // analogRead(A0)
int lecturaLdr = 0;     // analogRead(A1)

if (modoManual) {
  // Manual: pot decide
  if (lecturaPot > 700) {
    // Acción fuerte
  } else {
    // Acción suave
  }
} else {
  // Automático: LDR decide
  if (lecturaLdr > 800) {
    // Está oscuro → acción
  } else {
    // Hay luz → otra acción
  }
}

9) Ejercicios (para práctica)

Ejercicio 1: Rango “VIP” (AND)

Con un potenciómetro, enciende un LED solo cuando el valor esté entre 400 y 600. Pista: (v >= 400 && v <= 600).

Ejercicio 2: “Alarma” por dos condiciones (OR)

Enciende un LED si se cumple al menos una condición:

  • El botón está presionado
  • O el sensor análogo está por encima de un umbral

Ejercicio 3: Umbrales con histéresis (nivel pro)

Si el LED parpadea cerca del umbral, agrega dos umbrales: uno para encender y otro para apagar (histeresis). Es como un termostato: evita indecisión.

Idea final: Ifs no son “solo código”, son diseño de comportamiento. Un buen proyecto es una buena colección de decisiones… y una mala colección de excusas.

Error clásico

Confundir = con ==. == compara. = asigna. Uno pregunta, el otro dicta sentencia.

Tip de legibilidad

Si tu condición ya parece ensayo: guarda partes en variables booleanas con nombres claros.
