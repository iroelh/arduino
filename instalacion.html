Instalación: Arduino IDE, drivers y ESP32 en el IDE

Este paso define la experiencia del curso: si el IDE está bien instalado y tu computadora “ve” la placa, lo demás fluye. Si no, el LED parpadea… pero tus emociones también.

Descargar Arduino IDE (oficial) Guía oficial de instalación (Arduino Support)

1) Sitios oficiales que sí conviene usar

Para evitar instaladores “creativos” de terceros, aquí están los dos vínculos que te interesan para clase: el sitio oficial (información general y ecosistema) y el sitio de descarga del Arduino IDE.

  • Sitio oficial Arduino: arduino.cc
  • Descarga de Arduino IDE: arduino.cc/en/software
  • Documentación IDE 2: Arduino Docs (IDE 2)
Recomendación práctica: Para la mayoría de cursos, usa Arduino IDE 2. Mantén la instalación limpia, y evita tener 3 IDEs distintos “porque sí”. La computadora también necesita paz.

2) Instalación típica (paso a paso, sin drama)

La instalación “típica” es la que te deja listo para: abrir el IDE, seleccionar placa/puerto, subir un ejemplo, y ver comunicación en el monitor serial.

Windows (recomendado para laboratorios)
  • Descarga el instalador desde Arduino Software.
  • Ejecuta el instalador (idealmente con permisos de administrador si el equipo está restringido).
  • Acepta la instalación de drivers si Windows lo solicita (cuando aplica).
  • Abre Arduino IDE → conecta tu placa por USB.
  • Ve a Tools → Board y selecciona tu placa.
  • Ve a Tools → Port y selecciona el puerto COM que aparezca.
  • Prueba: File → Examples → 01.Basics → Blink y súbelo.

Si el puerto no aparece, casi siempre es: cable solo-carga, driver faltante, o puerto USB problemático.

macOS
  • Descarga Arduino IDE desde Arduino Software.
  • Instala/arrastra la app a Applications (según el paquete).
  • Conecta la placa y revisa permisos si el sistema pregunta.
  • Selecciona placa y puerto en Tools.

En macOS los clones pueden requerir drivers específicos (ver sección CH340).

Linux
  • Instala Arduino IDE desde el sitio oficial (evita repos muy viejos, si es posible).
  • Conecta la placa y verifica permisos del puerto serie (grupo dialout suele ser clave).
  • Selecciona placa/puerto y prueba Blink.

3) Driver CH340/CH341 (cuando tu “Arduino” es un clon)

Muchos Arduinos compatibles (y también algunas placas ESP32 económicas) usan un convertidor USB–Serial de la familia CH340/CH341. Cuando tu computadora no reconoce la placa (no aparece puerto), instalar este driver suele resolverlo.

Pista rápida: Si conectas la placa y no aparece ningún puerto nuevo, sospecha del cable o del driver. Si además la placa “se ve genérica” o es compatible, el CH340 entra a escena como protagonista.

Descarga oficial del driver (fabricante WCH): CH341SER (CH340/CH341) — WCH

Instalación CH340 en Windows (guía breve)
  • Descarga el instalador oficial CH341SER.EXE.
  • Ejecuta el instalador (si tu equipo es institucional, pide permisos de admin si hace falta).
  • Reinicia el IDE (y si algo se pone necio, reinicia Windows: sí, a veces funciona por arte oscuro corporativo).
  • Conecta la placa y vuelve a revisar Tools → Port.
¿Cómo sé si lo necesito?
  • Si el puerto no aparece en Arduino IDE.
  • Si en Windows, en “Administrador de dispositivos”, ves un dispositivo USB/Serial sin driver.
  • Si tu placa (Arduino compatible/ESP32 barata) anuncia “CH340/CH341” en la serigrafía o en la ficha del vendedor.

4) Programar ESP32 desde Arduino IDE (configuración oficial)

Para “controlar” (programar) una placa ESP32 desde Arduino IDE, no cambias el IDE: lo que haces es instalar un paquete de tarjetas (board support package) de Espressif, usando el Boards Manager.

Paquete ¿Cuándo usarlo? URL (para “Additional Board Manager URLs”)
Stable Clase normal y laboratorios: menos sorpresas. https://espressif.github.io/arduino-esp32/package_esp32_index.json
Development Solo si necesitas algo muy nuevo y aceptas riesgo. https://espressif.github.io/arduino-esp32/package_esp32_dev_index.json
Instalación ESP32 (Boards Manager, paso a paso)
  • Abre Arduino IDE.
  • Ve a File → Preferences.
  • En Additional Board Manager URLs, pega la URL de Stable: https://espressif.github.io/arduino-esp32/package_esp32_index.json
  • Da OK / Apply.
  • Ve a Tools → Board → Boards Manager.
  • Busca esp32 e instala “ESP32 by Espressif Systems”.
  • Reinicia Arduino IDE (recomendado).
  • Selecciona tu placa: Tools → Board (por ejemplo “ESP32 Dev Module” si es genérica).
  • Selecciona el puerto: Tools → Port.
  • Prueba con Blink o un ejemplo de ESP32.

Nota: algunas placas ESP32 requieren mantener presionado el botón BOOT durante el upload (depende del modelo/bootloader).

Troubleshooting rápido (lo típico en clase)
  • No aparece el puerto: prueba otro cable USB (muchos son solo carga), otro USB, o instala CH340 si aplica.
  • Upload falla: revisa que elegiste la placa correcta y el puerto correcto; intenta presionar BOOT al subir.
  • Se conecta y desconecta: cable malo o puerto USB con poca potencia.
  • Permisos (Linux): falta acceso al puerto serie (grupo dialout u equivalente).

5) Cierre: la prueba “mínima viable”

Si logras subir Blink a Arduino y ves un puerto estable; y además logras instalar el paquete de ESP32 (aunque no lo uses todavía), estás listo para el resto del curso.
